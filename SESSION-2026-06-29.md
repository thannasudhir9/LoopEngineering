### What Is Loop Engineering?

**Core definition (Addy Osmani):**
> "Loop engineering means replacing yourself as the person who prompts the agent. You design the system that does it instead."

**The fundamental shift:**
- **Before:** Human writes prompt → reads response → writes next prompt (manual, turn-by-turn)
- **After:** System discovers work → dispatches agents → verifies results → determines next steps autonomously

**Five core components:**

| Component | Purpose |
|-----------|---------|
| **Automations** | Scheduled discovery/triage — the heartbeat that makes it a true loop |
| **Worktrees** | Git-isolated dirs — parallel agents don't overwrite each other |
| **Skills** | Codified knowledge (SKILL.md) — context survives across runs |
| **Connectors** | MCP integrations — connects agents to real tools |
| **Sub-agents** | Maker vs Checker separation — writer ≠ reviewer |

**Loop types (ReAct pattern — Princeton/Google research):**
1. **Retry Loop** — simple pass/fail, atomic tasks
2. **Plan-Execute-Verify** — multi-step, order-dependent
3. **Explore-Narrow** — debugging unknown errors, tests multiple paths
4. **Human-in-the-Loop** — pauses for ambiguous requirements

**Key principles:**
1. Automation as Heartbeat — scheduled runs surface work continuously
2. Isolation via Worktrees — parallel agents need isolated dirs
3. Skills prevent Intent Debt — every session without skills starts cold
4. Maker/Checker Separation — agent that wrote code can't reliably grade it
5. Connectors enable real-world action — filesystem-only loops have narrow impact
6. Goal-based stopping — a separate model decides "done", not the one that did the work

## Architecture Design

### System Overview

### Component Breakdown

#### `src/state.py` — Central State Store
- **Purpose:** Single source of truth. All projects, tasks, loops stored here.
- **Persistence:** Writes to `data/state.json` on every mutation.
- **Push:** Maintains list of `asyncio.Queue` subscribers. Every mutation broadcasts an event.
- **Key functions:**
  - `upsert_project(folder, name)` — register a project folder
  - `create_task(project_id, title, description, priority, tags)` — new task
  - `update_task(task_id, **kwargs)` — patch any task field
  - `create_loop(task_id, goal, max_iterations)` — start a loop
  - `append_loop_log(loop_id, message, level)` — add iteration log entry
  - `finish_loop(loop_id, status)` — done | failed
  - `set_processes(procs)` — update live process list (ephemeral, not persisted)
  - `subscribe()` → asyncio.Queue — get a WebSocket broadcast channel
  - `get_snapshot()` — full state dump for new WebSocket connections

#### `src/monitor.py` — AI Process Scanner
- **Purpose:** Detect all AI tools running on the machine.
- **How:** Uses `psutil.process_iter()` to scan all processes every 5 seconds.
- **Detected tools:** Claude Desktop, Claude Code, Cursor, GitHub Copilot, Continue, Ollama, LM Studio, Aider, Cody
- **Deduplication:** Keeps highest-memory entry per label (avoids multi-process duplicates)
- **Key function:** `monitor_loop(interval=5.0)` — async background task, calls `state.set_processes()`

#### `src/watcher.py` — Filesystem Watcher
- **Purpose:** Watch registered project folders for file changes.
- **How:** Uses `watchdog.observers.Observer` with custom `FileSystemEventHandler`.
- **Events emitted:** `created`, `modified`, `deleted` — broadcast to all WebSocket subscribers
- **Threading bridge:** watchdog runs in a thread; uses `asyncio.run_coroutine_threadsafe` to push events to the async event loop
- **Key functions:**
  - `watch_project(project_id, folder, event_loop)` — start watching a folder
  - `stop_watcher()` — clean shutdown

#### `src/notes.py` — Notes Generator
- **Purpose:** Auto-create structured markdown docs per task.
- **Output structure:** `notes/<project-slug>/<date>-<task-slug>/`
  - `index.md` — task summary, links to all docs
  - `plan.md` — goal, scope, steps, risks, timeline
  - `design.md` — problem, solution, API contract, data model, error handling, alternatives
  - `architecture.md` — system context, component diagram, data flow, key decisions, NFRs
  - `loop-log.md` — each loop iteration appended as a new section
- **Key functions:**
  - `create_task_notes(project_name, task_id, task_title, description, tags)` → path
  - `append_loop_entry(notes_path, iteration, status, action, result, next_step)`
  - `update_task_status_in_notes(notes_path, status)`

#### `server.py` — FastAPI Backend
- **Purpose:** REST API + WebSocket server. Glues all modules together.
- **Startup (lifespan):** starts filesystem watcher, starts process monitor loop, re-registers existing projects
- **Static serving:** serves `dashboard/index.html` at `GET /`
- **WebSocket:** on connect sends full snapshot; then streams events from state queue

#### `dashboard/index.html` — Single-File Dashboard
- **Purpose:** Full web UI — no framework, no build step, vanilla JS.
- **Tech:** Pure HTML + CSS + JS in one file. Dark theme. WebSocket-driven.
- **Views:**
  - **Overview** — stats row, AI processes, running loops, high-priority tasks, recent activity
  - **AI Processes** — live process list with PID, CPU%, memory
  - **All Tasks** — full task table with status/priority/tags
  - **Loops** — all loops with progress bars and iteration logs
  - **Activity** — filesystem event feed
  - **Per-Project** — project stats + tasks + loops filtered by project
- **Modals:** Add Project, New Task, Start Loop
- **WebSocket:** auto-reconnects every 3s on disconnect; snapshots on connect; patches state on each event

## Using Loops From CLI / Other AI Agents

```bash
# 1. Add your project
curl -X POST http://localhost:7070/api/projects \
  -H 'Content-Type: application/json' \
  -d '{"folder":"/Users/sthanna/MyProject","name":"My Project"}'

# 2. Create a task
curl -X POST http://localhost:7070/api/tasks \
  -H 'Content-Type: application/json' \
  -d '{"project_id":"/Users/sthanna/MyProject","title":"Fix auth bug","priority":"high","tags":["bug","auth"]}'

# 3. Start a loop (use task id from step 2)
curl -X POST http://localhost:7070/api/loops \
  -H 'Content-Type: application/json' \
  -d '{"task_id":"<uuid>","goal":"All auth tests pass","max_iterations":10}'

# 4. Log each iteration
curl -X POST http://localhost:7070/api/loops/<loop_id>/log \
  -H 'Content-Type: application/json' \
  -d '{"message":"Iteration 1","action":"ran pytest","result":"3 failures","next_step":"fix token expiry"}'

# 5. Mark done
curl -X POST "http://localhost:7070/api/loops/<loop_id>/finish?status=done"
```
